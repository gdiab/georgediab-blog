# Hero image style prompt

Use this as the reusable base prompt for georgediab.com blog hero images. Adapt the `[TOPIC]` section for each post.

## Style notes

Based on recent hero images, the house style is:

- Editorial tech illustration, not a literal screenshot or stock image.
- Clean software-systems language: cards, code panes, nodes, arrows, checks, dashboards, pipelines.
- Wide 16:9 composition with a clear central focal point and generous negative space.
- Dark navy / midnight blue backgrounds, or pale blue-gray UI cards for cleaner diagram pieces.
- Warm amber/orange glow as the main accent, with teal/cyan and small green/yellow status accents.
- Soft depth: rounded rectangles, subtle shadows, gentle bloom, faint grids or dotted paths.
- Human figures, when used, should be anonymous, from behind or in profile, not posed.
- UI/code should be symbolic and mostly unreadable, using abstract lines rather than real text.

Avoid:

- Stock-photo realism, smiling corporate people, handshake imagery.
- Logos, brand names, real product screenshots, or copyrighted UI.
- Readable code, dense tiny text, cluttered dashboards.
- Bright rainbow palettes, excessive cyberpunk neon, mascot/cartoon style.
- Hyperrealistic faces or hands.

## Reusable prompt

Create a wide 16:9 editorial hero illustration for a technical software blog. Style should match a modern developer-focused publication: clean, cinematic, slightly painterly digital illustration with soft depth and atmospheric lighting.

Use a deep navy or blue-gray background. Build the scene around a clear central focal point related to: **[TOPIC]**.

Represent the concept using abstract software UI elements such as rounded cards, code panes made of unreadable short lines, flow arrows, nodes, status dots, checkmarks, server blocks, terminal panes, or pipeline diagrams. Keep all UI symbolic and non-readable.

Composition should be spacious and balanced, optimized for a blog hero image. Use either:

1. a dark cinematic scene with an anonymous developer viewed from behind or in profile, illuminated by warm glowing screens, or
2. a clean architecture diagram with floating rounded panels, subtle shadows, left-to-right flow, and a central module.

Color palette: midnight navy, slate blue, soft gray, off-white, warm amber/orange glow, teal/cyan accents, small green/yellow status dots. Add subtle gradients, soft shadows, gentle bloom, and minimal background texture such as faint grid lines, dotted paths, or brush-like shapes.

Mood: thoughtful, technical, polished, calm, and high-signal.

No branding, no logos, no readable text, no photorealistic stock-photo look, no clutter.

## Negative prompt

Readable text, real code, logos, brand names, stock photo, smiling corporate people, cluttered dashboard, excessive neon, cartoon mascot, hyperrealistic face, distorted hands, garish colors, dense tiny details, watermark.
